# pricing-versioning

## 目的

定义定价版本与规则版本的锁定与可审计快照（SKU snapshot），确保订单创建后计费规则稳定、可追溯，避免后续改价引发争议。

## 需求

### 需求: 订单创建锁定版本
每个订单创建时必须锁定：`pricing_version` 与 `rules_version`。

#### 场景: 后续改价不影响历史订单
- **当** 系统发布新定价版本
- **那么** 历史订单仍按其锁定版本计算费用

### 需求: SKU Snapshot 可审计
List Building SKU 的选择必须以 snapshot 形式锁定在订单上（可审计）。

#### 场景: 复核订单价格
- **当** Client 对价格提出疑问
- **那么** Admin 可查看订单的 SKU snapshot 与版本信息

### 需求: 支持 MQL 行业定价
若启用 MQL，系统必须支持按行业/规则版本的定价计算。

#### 场景: 不同行业同一 SKU 价格不同
- **当** 两个行业对应不同 MQL 定价
- **那么** 订单使用行业匹配的定价并锁定版本
