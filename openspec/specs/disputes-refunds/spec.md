# disputes-refunds

## 目的

定义交付后的争议与退款最小闭环：窗口、理由代码、证据链、HITL 仲裁留痕与退款方式，保证可审计与可执行。

## 需求

### 需求: 交付后 7 天争议/退款窗口 + 理由代码
系统必须支持交付后 7 天内提出争议/退款，并要求 reason codes。

#### 场景: 超过窗口拒绝
- **当** 已交付超过 7 天仍提交争议
- **那么** 系统拒绝并提示窗口规则

### 需求: 自动附带证据链
争议必须自动附带证据链（Qualification Report）。

#### 场景: 提交争议时自动附带报告
- **当** Client 提交 dispute
- **那么** 系统自动关联对应 Qualification Report

### 需求: Admin/HITL 仲裁留痕
Admin/HITL 仲裁必须留痕（可审计）。

#### 场景: 仲裁通过并退款
- **当** Admin 判定争议成立并执行退款
- **那么** 仲裁记录包含决策与证据引用

### 需求: 优先返还 credits，必要时原路退款
系统必须优先返还 credits；必要时原路退款；账本必须可审计。

#### 场景: 优先返还 credits
- **当** 争议成立
- **那么** 系统优先返还 credits 并记录账本 refund 事件
