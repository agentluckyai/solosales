# payments-stripe

## 目的

定义 MVP 的 Stripe Checkout + 3DS 支付能力与账户前置条件，保证支付链路可用且可审计。

## 需求

### 需求: Stripe Checkout + 3DS
系统必须支持 Stripe Checkout，并支持 3DS。

#### 场景: 3DS 支付
- **当** 银行要求 3DS
- **那么** Client 可完成支付并回写充值/订单状态

### 需求: 下单/支付前置条件
系统必须禁止未注册或未邮箱验证的用户下单/支付；并且必须收集账单信息。

#### 场景: 未邮箱验证禁止支付
- **当** 用户未完成邮箱验证但尝试进入 checkout
- **那么** 系统拒绝并引导完成验证

### 需求: 首单风控额度可配置
系统必须支持可配置的首单风控额度（或等价风控策略）。

#### 场景: 首单超限
- **当** 首单金额超过配置上限
- **那么** 系统阻止并记录风控原因
