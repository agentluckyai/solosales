# auth-permissions

## 新增需求

### 需求: 角色与权限边界（MVP）
系统必须至少支持两类角色，并强制其权限边界：
- Client：充值/下单/查看进度/下载/争议
- Admin/HITL：规则配置/抽检/仲裁/停发

#### 场景: Client 无法访问 Admin 能力
- Given: Client 登录
- When: 访问 Admin-only API
- Then: 系统必须拒绝（403）并记录审计

### 需求: 注册 + 邮箱验证为下单/支付前置条件
系统必须禁止未注册或未邮箱验证的用户下单/支付。

#### 场景: 未验证用户下单被拦截
- Given: 用户已注册但未验证邮箱
- When: 创建订单
- Then: 系统拒绝并引导完成验证
